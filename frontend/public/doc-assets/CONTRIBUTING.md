# Contributing

Update docs and ADRs with major changes.

Use `docs/PROJECT_TRAIL.md` as the development closeout checklist for context
that should survive beyond chat history.

- Put durable architectural decisions in `docs/adr/`.
- Put point-in-time ADR artifacts in `docs/adr/assets/{id}-{slug}/`.
- Put curated feature or research conversation summaries in `docs/context/`.
- Put early change requests and feature briefs in `docs/requests/`.
- Put operating policies in `docs/policies/`.

Put explanatory visuals next to the relevant prose or workflow. Keep
`## Resources` for supporting artifacts, provenance, and links rather than a
catch-all gallery of diagrams.

At milestone boundaries, review trail notes for ADR candidates, stale requests,
missing artifact links, and policy changes that need clearer ownership.
